# /keisai：掲載通知照合（5ステップ自動実行）

スプレッドシートとSlack通知を貼り付けるだけで、5ステップの照合を一括実行します。

## 使い方

```
/keisai
■ スプレッドシート（会社名・プラン・担当）
企業名	求人登録	プラン	ランク	制作担当
（表をそのまま貼り付け）

■ Slack通知
（Slackのテキストをそのまま貼り付け）
```

---

## 受け取った情報

$ARGUMENTS

---

## このコマンドの成果物（最重要）

**このコマンドの成果物は「出力フォーマット」のすべての表である。**

- 各ステップは「分析」だけで終わらせず、**そのステップの表を直後に必ず出力する**こと。分析結果を最後にまとめて出力しようとしない。
- Bash/Python の実行結果（生ログ）を出して終わりにしない。必ず指定の表形式に整形して出力する。
- **完了条件：Step 1〜5 の5つの表と「⚠ 要確認まとめ」の表をすべて出力し終えるまで、ターンを終了してはいけない。** 途中で「分析が終わった」と判断して止めない。
- 問題がない項目も省略しない（「✓」「異常なし」と明記する）。

---

## 実行手順

### 準備：入力データの解析

$ARGUMENTS から以下を抽出する：
1. 「■ スプレッドシート」以降のタブ区切りデータ → 会社リスト（企業名・求人ID・プラン・ランク・担当）
2. 「■ Slack通知」以降のテキスト → Slack通知リスト

CSVフォルダ（固定）: `C:\Users\chiba\Documents\掲載通知\`
- `jp_job.csv` … 求人データ（Shift-JIS = cp932）
- `jp_company_schedule.csv` … 掲載スケジュール（UTF-8）
- `jp_contract.csv` … 契約プラン一覧（Shift-JIS = cp932）
- `jp_staff_company.csv` … 企業スタッフ紐づけ（Shift-JIS = cp932）：company_company_id × company_staff_id で照合

**CSVの読み込み・検索・照合はすべて確認不要で進める。新規作成・上書き・削除する場合のみ確認する。**

各ステップは「① Bash/Python で照合 → ② その場で表を出力」の順で進める。**②の表を出さずに次のステップへ進まない。**

---

### Step 1：求人データ確認（jp_job.csv）→ 確認後すぐに ✅Step1 の表を出力

Bash で Python を使い、以下を実行する：

```python

# スプレッドシートから抽出した全求人IDリスト（「追加」行も含む）
target_ids = [...]

# jp_job.csv を cp932 で読み込み
# 対象IDごとに job_id, job_company_id, job_publish_time, job_conf_publish を取得

```

確認する内容：
- 全求人IDがCSVに存在するか
- 掲載日時（job_publish_time）が設定されているか
- 掲載承認（job_conf_publish = 1）になっているか

→ 確認できたら、ただちに下記「✅ Step 1」の表を出力する。

---

### Step 2：スケジュール確認（jp_company_schedule.csv）→ 確認後すぐに ✅Step2 の表を出力

Step 1 で得た company_id ごとに jp_company_schedule.csv（UTF-8）を検索する。

確認する内容：
- 各社の掲載スケジュールが登録されているか
- 掲載日時（schedule_time）が正しいか
- 紐づく contract_id を取得（Step 3 で使用）
- 「追加」のみの会社はスケジュール登録不要（メイン求人と同じ company_id で登録済み）

→ 確認できたら、ただちに下記「✅ Step 2」の表を出力する。

---

### Step 3：プラン照合（jp_contract.csv）→ 確認後すぐに ✅Step3 の表を出力

Step 2 で得た contract_id ごとに jp_contract.csv（cp932）を検索する。

確認する内容：
- contract_name（プラン名）がスプレッドシート記載と一致するか
  - 回数（1/4 など）・週数・+365 の有無を個別に確認
- contract_company_conf_rank（ランク）がスプレッドシートのランクと一致するか

→ 確認できたら、ただちに下記「✅ Step 3」の表を出力する。

---

### Step 4：Slack照合 → 確認後すぐに ✅Step4 の表を出力

Slack通知テキストを解析し、以下を確認する：

- スプレッドシートの全社がSlackに報告されているか（漏れがないか）
- プラン名・回数がスプレッドシートと一致しているか（+365 の有無も含む）
- 同じ会社が重複報告されている場合は重複として記録する
- Slack報告にあってスプレッドシートにない会社も記録する

→ 確認できたら、ただちに下記「✅ Step 4」の表を出力する。

---

### Step 5：スタッフ登録確認（jp_staff_company.csv）→ 確認後すぐに ✅Step5 の表を出力

Step 1 で得た company_id をキーに jp_staff_company.csv（cp932）を検索する。

```python

# company_staff_id の件数を集計
# company_spool_status = 0 のものは無効アカウントとして件数から除外
```

確認する内容：
- 各社に1件以上の有効なスタッフアカウントが紐づいているか（spool_status=1 のみカウント）
- 未登録の場合は「企業への連絡・登録依頼が必要」として記録する
- ID:12303・13476・13877 はちばキャリ社内管理アカウントのため、表示は「ほか」とまとめる

→ 確認できたら、ただちに下記「✅ Step 5」の表を出力する。
